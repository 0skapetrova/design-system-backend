# Обзор схемы БД (schema overview)

## 1. Цели схемы
Коротко: что за система и какие задачи решает (пользователи, проекты, клиенты, спецификации и т.п.).

## 2. Основные сущности (MVP)
- `users` — зарегистрированные пользователи системы.
- `phone_verifications` — одноразовые коды подтверждения телефона.

## 3. Связи между сущностями
- `phone_verifications.user_id → users.id` (опциональная связь после регистрации).

## 4. Планируемые сущности
- `projects` — проекты интерьеров.
- `clients` — клиенты.
- `specs`, `items`, `spec_versions` — спецификации и позиции.

## 5. Принципы проектирования
- UUID как PK.
- timestamptz для временных полей.
- soft-delete через `deleted_at` там, где нужно архивирование.
